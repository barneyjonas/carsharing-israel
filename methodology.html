How this overview is built

Methodology

This page explains what the MVP includes, what it excludes, and which fields need further verification before the site should make stronger claims.

Inclusion rule

The MVP includes services that provide app-based or self-service access to cars for short-term use in Israel. It includes classic carsharing and borderline short-term access services when they solve the same user problem.

Exclusion rule

Ordinary branch-based car rental is excluded unless the service has a clear self-service, app-based, hourly, area-based or carsharing-like access model.

Classification

Providers are grouped into round-trip carsharing, free-floating / area-based carsharing, self-service short-term app rental, and brand-based short-term access.

Verification status

Each provider has a confidence level and a last-checked field. This indicates how strong the current public-data basis is, not whether every detail has been independently verified.

Fields needing verification

  • Exact city and station coverage
  • Tourist eligibility
  • Israeli ID and licence requirements
  • Credit-card and deposit rules
  • Insurance and deductible conditions
  • Shabbat and holiday availability
  • Current pricing and fees

Future trust layer

Later versions can add labels such as “provider confirmed”, “last verified”, “pricing checked”, and “city coverage checked” to avoid overclaiming and to build user trust.
